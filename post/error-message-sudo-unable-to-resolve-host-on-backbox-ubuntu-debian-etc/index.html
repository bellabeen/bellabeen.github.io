Muhammad Bella Buay Nunyai | 04-04-2018

Error message 'sudo: unable to resolve host ' on backbox, ubuntu, debian etc

#Linux #Tutorial #2018 #Ubuntu #2019 #Info #Networking #Internet #Trick #Analisis #Backbox

In a few days ago when after installing linux backbox on personal pc happened a strange error
code error

    Error message ‘sudo: unable to resolve host <USER>‘After the analysis on the day before I changed the name of hostname on my linux, because it replace the hostname error occurs on my linux when accessing the root. There is a difference between hostname and hosts on my linux
We make sure the names on the hostname and host happen in common


    hachi @ geek: / etc / network $ cat / etc / hostname

    geek

    hachi @ geek: / etc $ cat hosts

    127.0.0.1 localhost

    127.0.1.1 geekWe can edit the hostname with the command

    hachi @ geek: / etc $ sudo nano / etc / hostnameWe can edit the host with the command

    hachi @ geek: / etc $ sudo nano / etc / hosts127.0.0.1 localhost127.0.1.1 geek# The following lines are desirable for IPv6 capable hosts:: 1 ip6-localhost ip6-loopbackfe00 :: 0 ip6-localnetff00 :: 0 ip6-mcastprefixff02 :: 1 ip6-allnodesff02 :: 2 ip6-allrouters
The code can be used on ubuntu, debian, backbox, linux time etc.So my brief tutorial may be useful

Fitur keamanan baru di OS Android

October 29, 2017


Layanan Internet Service Provider (ISP) Anda, atau hacker tingkat jaringan tidak dapat memata-matai komunikasi https.
Tapi tahukah Anda - ISP masih bisa melihat semua permintaan DNS Anda, yang memungkinkan mereka mengetahui situs web yang Anda kunjungi

Google sedang mengerjakan fitur keamanan baru untuk Android yang dapat mencegah lalu lintas Internet Anda dari serangan spoofing jaringan.
Hampir setiap aktivitas Internet dimulai dengan permintaan DNS, menjadikannya blok bangunan fundamental dari Internet. DNS berfungsi sebagai buku telepon Internet yang bisa menyelesaikan alamat web yang mudah dibaca manusia, seperti thehackernews.com, terhadap alamat IP mereka.
Permintaan dan tanggapan DNS dikirim dalam bentuk teks yang jelas (menggunakan UDP atau TCP) tanpa enkripsi, yang membuatnya rentan terhadap menguping dan membahayakan privasi.
ISP secara default menyelesaikan permintaan DNS dari server mereka. Jadi saat Anda mengetikkan nama situs web di browser Anda, kueri pertama masuk ke server DNS mereka untuk menemukan alamat IP situs web, yang akhirnya memaparkan informasi ini (metadata) ke ISP Anda.
Selain itu, DNS Security Extensions - dikenal luas sebagai DNSSEC - hanya menawarkan integritas data, bukan privasi.

Sama seperti protokol terenkripsi Transport Layer Security (TLS) yang mengamankan koneksi HTTPS secara kriptografis, DNS-over-TLS secara dramatis meningkatkan privasi dan keamanan dengan pencarian DNS ter-end-to-end yang otentik.
Google dilaporkan menambahkan dukungan “DNS over TLS” ke Android Open Source Project (AOSP), saat ini dalam tahap percobaan, untuk memungkinkan pengguna smartphone mengaktifkan atau menonaktifkan fitur “DNS over TLS” di bawah pengaturan Opsi Pengembang.

Indikasi Nama Server (SNI) - perpanjangan protokol TLS - juga menunjukkan ISP yang nama hostnya dihubungi oleh browser pada awal proses ‘jabat tangan’.
Jadi untuk menikmati anonimitas penuh, pengguna masih diharuskan menggunakan layanan VPN aman tepercaya yang dikombinasikan dengan protokol DNS-over-TLS.
